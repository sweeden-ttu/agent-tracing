Experimental Setup

Benchmark

We instantiate the architecture on the Rogii Wellbore Geology
Prediction task hosted on Kaggle
(rogii-wellbore-geology-prediction).  The competition is a
multi-well regression benchmark: agents predict True Vertical
Thickness (TVT) lithology labels at each test depth index from
horizontal-well log features, typewell GR curves, and optional
geology-surface columns.  The public leaderboard ranks teams by
RMSE on the post-perforation-start (post-PS) subset; submission
validity requires an id column plus a tvt prediction column
matching sample_submission.csv exactly.

The task is selected for three reasons: it requires non-trivial
feature engineering (well-log normalization, typewell GR alignment,
GroupKFold by well, PS-point leakage masks, formation/spatial
features); it has a clearly defined submission envelope (id, tvt,
row count, no NaNs); and its leaderboard is active enough that a
verified trace-language pipeline constitutes a reproducible signal.
Six preprocessing-variant trace languages under
examples/rogii/traces/preprocessing/{variant}/trace_language.csv
instantiate distinct scientific concepts on this benchmark; see
Section sec:trace-csv for the directory audit and per-variant
critical review against BASE_PAPERS.md.

Hardware and software environment

itemize
Hardware: a single laptop (the author's macOS development
machine).  CPU and RAM bounds are recorded in the run log; both
are finite and the realised computation is therefore an LBA in
the formal sense of Section sec:preliminaries.
Operating system: macOS, with the Kaggle CLI version locked
in the run-time configuration.
LLM backbone: Google Gemini, accessed through the
google-genai client.  Two model strings appear in the
realised trace: gemini-2.5-pro for Type-0 deliberation
steps; gemini-2.5-flash and
gemini-2.5-flash-lite for finite-state housekeeping and
verification-adjacent steps.
Embedding backbone: text-embedding-004 for the
research-agent vector store and the tools-index keyword
embedding.
itemize

The deliberate restriction of the backbone to a single hosted LLM
family is methodological: the goal is to minimise the
architectural surface area while exhibiting the trace-language
phenomenon.  We report no comparison across backbone families.

Probes and predictions

To preserve falsifiability, we pre-register four probes against
the architecture before running the binding.

P1 (Type-3 envelope of ).
The verifier's trace language is regular.  The empirical
prediction is that the membership decision per candidate prefix
runs in time linear in prefix length, with a flat constant in
prefix length.  Refutation: any super-linear timing on
's side.

P2 (Type-0 envelope of ).
The Goal-Driven agent emits at least one trace whose length
strictly exceeds the LBA bound that any of the start-language
columns would predict.  Refutation: the realised trace is no
longer than the start-language columns.

P3 (Emergent thirteenth agent).
The realised trace contains operation symbols that are not in, and these symbols cluster into a thirteenth column
(slurm_episodic_trainer) absent from
examples/rogii/traces/preprocessing/baseline_column_transformer/trace_language.csv (start language L_0).  Refutation: no new
column appears, or the new symbols do not cluster into a coherent
sub-agent.

P4 (Leaderboard placement).
The verified trace, executed end-to-end via slurm_full_pipeline, is
predicted to yield a public leaderboard placement at or above the top
tertile of submissions (rank <= ceil(N/3) for N teams on the public
leaderboard snapshot).  Refutation: placement strictly worse than the
tertile threshold.  As of May 2026 this probe is pre-registered but
not yet verified; numerics are reported in Section sec:results as
TBD until the Slurm gate completes.

These four probes are pre-registered in the sense that their
predictions are stated before the binding is run.  We report
their outcomes in Section sec:results.
