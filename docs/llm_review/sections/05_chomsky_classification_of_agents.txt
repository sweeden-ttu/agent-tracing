Chomsky Classification of Agents

We now classify trace languages by Chomsky type and exhibit, for
each type, the canonical agent shape from the empirical
Rogii Wellbore Geology Prediction system.  The classification is the
literal one of Section sec:preliminaries; there is no
new mathematics here, only the application of an old hierarchy to
a new object.  The novelty, if any, is the identification of
which sub-agents in the empirical CSVs inhabit which class.

Type-3: regular trace languages

definition[Type-3 agent]
An agent  is Type-3 if, equivalently, if there exists a finite
automaton  with.
definition

In the empirical system, the Type-3 agents are
toolsindex and unittestrunner.  Both have
trace languages over a small finite alphabet whose strings are
generated by a finite-state controller:
itemize
toolsindex: the trace language is the regular
expression

(loadschema)(composetooltext)(embedtooltext)(scoreagainstindex)

read straight off the column.  No back-edge appears.  No stack
discipline is required.
unittestrunner: the trace is a sequence of
test* keywords drawn from the finite test catalogue.
The deciding finite automaton is the Aho-Corasick DFA over the
catalogue aho1975corasick.
itemize

The Data-Driven verifier  of this paper is realised
as the union of these two columns, compiled into a single
Aho-Corasick DFA.  Membership in  is decided in
 time and  space per candidate trace, by Rabin and
Scott's classical
construction rabin1959finite,aho1975corasick.

Type-2: context-free trace languages

definition[Type-2 agent]
An agent  is Type-2 if, equivalently, if there
exists a pushdown automaton  with  and
 is not regular.
definition

In the empirical system, the Type-2 sub-agents are
planneragent and revieweragent.  Both
exhibit nested structure that no finite automaton can recognise.
The planner emits a step-list whose nesting depth is determined
by the EDA bundle:
verbatim
emit_step(id; title; instructions; tools)
  -> emit_step(...) ->...
emit_success_criteria
emit_plan_json
verbatim
and the reviewer's score-and-clamp loop pushes a configuration
onto an implicit stack each time a sub-test fails:
verbatim
collect_failed_tests -> compose_reviewer_prompt
  -> clamp_score_to_2_on_unit_failure
  -> clamp_score_to_1_5_range
  -> emit_review_result(...).
verbatim
The stack discipline is implicit but recoverable: the trace
language of planneragent matches Dyck-language
shapehopcroft2006automata when one identifies opening
composeplannerprompt and closing
emitplanjson as bracket symbols.

The indexed-grammar zone (between Type-2 and Type-1)

Aho's indexed grammars aho1968indexed occupy a class
 strictly between
and.  Their distinguishing feature is a finite
stack of indices attached to each nonterminal during a
derivation.

In the empirical system, the indexed-zone sub-agent is
soporchestrator, whose trace language exhibits a
finite-index repair stack: the tokens
capiterationsatmax(=3) and
retryphaseonlowscore together form an indexed
loop whose nesting is bounded by the constant 3.  An indexed
grammar with three index symbols recognises this loop; a pushdown
automaton without the index discipline cannot, because the loop
exhibits cross-serial dependencies between the iteration counter
and the phase counter.  We do not enlarge the formal hierarchy;
we observe that the indexed-zone class is the natural home of
this column.

Type-1: context-sensitive trace languages

definition[Type-1 agent]
An agent  is Type-1 if,
equivalently, if there exists a linear-bounded automaton
with  and  is not in the
indexed class.
definition

The Type-1 sub-agents are taskorchestrator and
kernelpipeline.  Their trace languages exhibit
linear-bounded workspace growth: each step's emitted string is
bounded by a constant times the input length, and the
computedependencygraph(prevstagetonextstage)
operation is, by construction, an LBA-style traversal of the
phase graph that visits each cell at most a constant number of
times.

The kernelpipeline's trace contains
generatekernelpackages(4copies),
writepackagestodisk, and
sequentialpushonvm(stageorder) - a strictly
linear workspace expansion in the number of stages.  No string
in  exceeds a constant multiple of
the input description's length, which is the LBA hallmark.

Type-0: recursively enumerable trace languages

definition[Type-0 agent]
An agent  is Type-0 if, equivalently, if there
exists a Turing machine  with  and
 is not context-sensitive.
definition

The Type-0 agents are model_trainer, the
developeragent (in its full unbounded mode of arbitrary
LLM code-generation), and the slurm_episodic_trainer.  The
slurm_episodic_trainer column is the canonical witness: it
contains a strictly unbounded enumeration of training operations
bootstrapindices, fitmodelonsubset,
scoretrainandval, accumulateoofpredictions,
trackbestepisodebyvalscore, parameterised by
loop bounds (foreachfoldforeachepisode) that
are themselves outputs of the agent's own deliberation.

The Goal-Driven agent of the architecture is the union of
these Type-0 sub-agents under the orchestration relation  given
in Section sec:methodology.  Its trace language is, by
Theorem thm:closure run in reverse, undecidable to verify
absent the Type-3 verifier - which is the
Robinson-MRDP robinson1949definability,davis1961decision
shadow of the Halting Problem in our setting.
