Recursive enumeration and undecidability

A language  is recursively enumerable
(r.e., or semi-decidable) if there exists a Turing machine
 such that, started on the empty input,  prints exactly the
strings of  on its output tape (in some order, possibly with
repetitions).  Equivalently,  is r.e. iff there exists a Turing
machine  that halts and accepts on input  iff,
and may run forever on.

A language  is decidable (recursive) if both
and its complement  are r.e.; equivalently, if there
exists a Turing machine that halts on every input and accepts iff.  The strict inclusion of decidable r.e. languages
inside r.e. languages is a theorem of
Turing turing1936computable; the undecidability of the
membership problem for arbitrary r.e. languages is the
Halting Problem, also Turing's.

A celebrated and non-elementary characterisation of the r.e. class
is Robinson's:  the r.e. subsets of  are exactly the
Diophantine subsets robinson1949definability,davis1961decision.
The corollary - that there is no algorithm deciding membership
in an arbitrary r.e. set defined by Diophantine constraints -
is the form in which we use the result.  An idealised Type-0
agent therefore inherits, by the same line of argument, the
property that no algorithm decides whether a given trace is
in the agent's trace language.

Idealised versus physically bounded machines

Every physical computation is bounded: the tape is a finite array
of cells, the time is finite, the energy is finite.  The
idealisation we adopt is the standard one of
Hartmanis and Stearns hartmanis1965computational: we
analyse machines whose tape and time are unbounded in the formal
specification, while observing that any concrete execution
witnesses only a finite prefix of the idealised computation.

For a Turing machine, fix a tape bound  and a
time bound.  The bounded machine  is the

most  cells in at most  steps.  Plainly  is a finite

Lemma lem:bounded-soundness is the formal warrant for the
"idealised Type-0" framing in
Section sec:methodology.  The Goal-Driven agent, as
implemented on the author's laptop, is a very large but finite
LBA.  For the analysis we treat it as an idealised TM; any trace
the bounded machine emits is also emitted by the idealised
machine, and the verification claims we discharge against the
idealised machine apply a fortiori to the bounded one.

Closure properties

We will need the following standard
results hopcroft2006automata:
enumerate
Regular languages are closed under union, intersection,
complement, concatenation, and Kleene star.
Context-free languages are closed under union, concatenation,
and Kleene star, but not under intersection or complement;
however, the intersection of a context-free language with a
regular language is context-free.
Context-sensitive languages are closed under union,
intersection, and concatenation; closure under complement was
established later (Immerman-Szelepcsenyi, beyond our citation
discipline; we use only union and intersection in this paper).
Recursively enumerable languages are closed under union,
intersection, and concatenation; intersection with a regular
language is r.e.
enumerate

The closure of r.e. under intersection with regular languages is
the closure that drives Theorem thm:closure: the verified
trace language - the intersection of the Goal-Driven Type-0
agent's trace language with the Data-Driven Type-3 verifier's
trace language - is r.e., but its membership question is
decided by the verifier's DFA whenever a trace is emitted
to the disk.  Verification cannot exceed Type-3.
